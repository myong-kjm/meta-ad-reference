# CLAUDE.md — 클로드 코드에게 주는 안내 (이 파일은 클로드 코드가 자동으로 읽어요)

> 이 파일이 있는 이유: 클로드 코드가 프로젝트 구조를 **다시 뒤져보지 않게** 미리 다 알려주기 위해서예요.
> 안 뒤져봐도 되니 = 토큰(=돈)이 적게 들고 = 답이 빨라집니다.

## 토큰 절약 규칙 (클로드 코드는 이걸 지켜주세요)

- 이 프로젝트에서 **사람이 손댈 곳은 딱 2곳**입니다 (아래 "같이 만들 곳"). 그 외 파일은 이미 완성되어 있으니 함부로 고치지 마세요.
- 작업 전에 **파일 전체를 다 읽지 마세요.** 아래 "파일 지도"에 위치가 적혀 있으니, 고칠 함수 1개만 열어서 보세요.
- 답변은 **짧게.** 긴 설명·요약 반복 금지. 바뀐 부분만 보여주고 끝.
- 새 라이브러리 설치·새 파일 생성은 **꼭 필요할 때만.** 가능하면 기존 함수 하나만 채우세요.
- 한국어로, 비개발자가 알아듣는 말로 설명해 주세요.

## 이 프로그램이 하는 일 (한 줄)

경쟁사(예: 클래스101)의 **페이스북/인스타 광고를 자동으로 모아서**, 내 컴퓨터의 보기 좋은 화면(대시보드)에서 훑어보는 도구. 외부 결제 0원, 내 컴퓨터에서만 돌아감.

## 파일 지도 (어디에 뭐가 있나)

```
dashboard/app.py        화면(대시보드). Streamlit으로 만든 보기 화면.
scraper/main.py         수집 전체 흐름의 '지휘자'. (이미 완성)
scraper/url_builder.py  광고 라이브러리 주소 만들기. (이미 완성)
scraper/extractor.py    받아온 응답에서 광고 알맹이 뽑기.  ← 같이 만들 곳 2
scraper/browser.py      사람처럼 보이게 브라우저 띄우기(봇 회피). (이미 완성)
scraper/human_like.py   랜덤 대기·스크롤 등 사람 흉내. (이미 완성)
scraper/media_download.py 이미지/영상 내려받기. (이미 완성)
scraper/db.py           수집한 광고를 저장/조회 (SQLite). (이미 완성)
scraper/notion_sync.py  노션에 자동 정리 (선택). (이미 완성)
data/ads.db             연습용 샘플 광고 40개가 들어있는 저장소.
data/samples/           보너스 단계 오프라인 테스트용 샘플 1건.
```

## 같이 만들 곳 (사람이 채우는 2곳)

### 1) [필수] `dashboard/app.py` 의 `apply_filters()` 함수
- 지금은 받은 광고를 **그대로 다 돌려주는** 빈 상태 (`return ads`).
- 그래서 대시보드는 잘 켜지지만, 사이드바 필터(활성/영상/최소 게재일)가 동작을 안 함.
- 할 일: 조건(활성 여부·이미지/영상·최소 게재일·날짜 범위)에 맞는 광고만 골라서 돌려주기.
- 확인법: 대시보드 사이드바 슬라이더를 움직이면 목록 개수가 줄어들면 성공.

### 2) [보너스/심화] `scraper/extractor.py` 의 `_node_to_ad()` 함수
- 지금은 광고 번호와 페이지 이름만 뽑고, 캡션·소재·게재일은 비어 있음.
- 할 일: `snapshot`/`node` 안에서 caption·media_urls·start_date·active_days·platforms 등을 뽑아내기.
- 확인법(인터넷 없이): `python scripts/test_extractor.py` → ❌가 ⭕로 바뀌면 성공.

## 자주 쓰는 명령

```bash
# 대시보드 켜기
streamlit run dashboard/app.py

# 보너스 단계 오프라인 테스트
python scripts/test_extractor.py
```

자세한 단계별 안내와 '복사해서 붙여넣을 프롬프트'는 `워크샵-가이드.md`에 있어요.
