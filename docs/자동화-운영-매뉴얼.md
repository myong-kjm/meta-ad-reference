# 광고 레퍼런스 자동 수집기 (Pro)

> **매일 오전 9시에 경쟁사 페이스북 광고를 자동으로 긁어서, 본인 컴퓨터의 Streamlit 대시보드에서 본다.**
>
> 영상·이미지·캡션·랜딩 URL 전부 수집, 어제 신규/지난 7일 활성 자동 분류.

[1주차 강의 기본 버전](../README.md)의 응용편입니다. 기본 버전(URL 수동 붙여넣기)을 먼저 익힌 다음에 이 자동화 버전으로 오세요.

---

## ⚠️ 시작 전 솔직한 안내

1. **이 도구는 본인 컴퓨터에서만 돌립니다.** Vercel/AWS 같은 클라우드 IP는 메타가 거의 다 차단해놔서 안 됩니다.
2. **봇 차단 확률을 최소화한 것**이지 0%는 아닙니다. 어쩌다 차단되면 24시간 쉬고 본인 Chrome으로 광고 라이브러리 정상 로그인 한 번 했다가 다시 시도하세요.
3. **메타가 광고 라이브러리 HTML/GraphQL 구조를 바꾸면 셀렉터가 깨집니다.** 1~3개월에 한 번 정도 빈도. 그땐 클로드 코드에 "scraper/extractor.py가 빈 광고를 뽑아오는데 응답 구조 다시 봐줘"라고 시키면 됩니다.
4. **하루 1회만 실행하세요.** 자주 돌리면 봇 시그널 됩니다.
5. 공개된 광고만 수집합니다. 로그인 강제 없음. 학습/연구 목적.

## 1. 처음 한 번만 — 설치

### 1.1 파이썬 + 의존성

```bash
cd (다운받은 폴더로 이동)

# 가상환경 만들기 (선택이지만 추천)
python3 -m venv .venv
source .venv/bin/activate

# 의존성 설치
pip install -r requirements.txt

# Playwright 브라우저 다운로드
playwright install chromium
```

### 1.2 본인 경쟁사 설정

```bash
cp config.example.json config.json
```

`config.json`을 열어서 `competitors` 배열을 채웁니다.

**`page_id` 얻는 법** (이게 핵심):
1. https://www.facebook.com/ads/library 접속, 국가 = 한국
2. 경쟁사 이름 검색 → 페이지 선택 → "이 광고주의 모든 광고 보기"
3. 브라우저 주소창 URL에서 `view_all_page_id=` 다음 숫자 복사
4. `config.json`의 `page_id`에 붙여넣기

예시:
```json
{
  "competitors": [
    {"page_id": "1947160905536967", "page_name": "클래스101", "notes": "온라인 클래스 경쟁사"},
    {"page_id": "123456789012345", "page_name": "탈잉", "notes": "직접 경쟁"}
  ]
}
```

### 1.3 Google Sheet 업로드 설정

수집한 광고 소재 경로, 캡션, 게재일, 활성 상태는 스크래핑 후 실습 시트에 자동 동기화됩니다.

기본 실습 시트:
https://docs.google.com/spreadsheets/d/본인-시트-ID/edit

`config.json`의 `google_sheet.credentials_path`에 Google 서비스 계정 JSON 파일 경로를 넣고, 해당 서비스 계정 이메일을 위 시트에 편집자로 공유하세요.

```json
"google_sheet": {
  "enabled": true,
  "spreadsheet_url": "https://docs.google.com/spreadsheets/d/본인-시트-ID/edit",
  "worksheet_name": "ads",
  "credentials_path": "service-account.json"
}
```

### 1.4 (중요) 본인 Chrome 프로필 한 번 준비

이 도구는 **본인 Chrome의 로그인된 세션을 빌려와서** 사람처럼 보이게 합니다.

- Chrome을 평소처럼 쓰시면 됩니다. 페이스북에 로그인되어 있어야 합니다.
- 첫 실행 때 도구가 본인 Chrome 프로필을 임시 폴더로 복사합니다 (원본은 안 건드림).
- macOS 기본 경로: `~/Library/Application Support/Google/Chrome/Default`

## 2. 매일 쓰는 법

### 수동 실행 (테스트 또는 즉시 수집)

```bash
# 신규 순으로 한 번 (= 어제 새로 올라온 광고 찾기)
python scraper/main.py --mode relevancy_monthly_grouped

# 노출수 순으로 한 번 (= 위너 후보 찾기)
python scraper/main.py --mode total_impressions

# 둘 다 (매일 자동 실행에서 쓰는 옵션)
python scraper/main.py --mode both
```

### 대시보드 켜기

```bash
streamlit run dashboard/app.py
```

브라우저가 자동으로 열립니다. 안 열리면 `http://localhost:8501`.

탭 5개:
- **📅 어제 신규** — 어제 우리 DB에 처음 등록된 광고
- **🔥 지난 7일 활성** — 7일 이상 살아있는 활성 광고 = 위너 후보
- **🏢 페이지별 비교** — 경쟁사 한 곳 깊게 보기
- **⭐ 위너 후보** — 마킹한 광고 모음
- **⚙️ 페이지 관리** — 경쟁사 추가/조회

## 3. 매일 9시 자동 실행 (macOS launchd)

```bash
./scripts/install_schedule.sh
```

끝. 이게 매일 오전 9시에 `scraper/main.py --mode both`를 알아서 실행합니다.

### 확인

```bash
launchctl list | grep halice
```

`com.halice.ad-scraper`가 보이면 등록 OK.

### 수동으로 한 번 트리거 (테스트용)

```bash
launchctl start com.halice.ad-scraper
```

### 로그 보기

```bash
tail -f data/logs/$(date +%Y-%m-%d)_cron.log
```

### 제거

```bash
launchctl unload ~/Library/LaunchAgents/com.halice.ad-scraper.plist
```

## 5. 봇 차단 의심 시 복구법

증상:
- 로그에 `🛑 봇 차단 감지` 메시지
- 페이지가 빈 채로 로드되거나 로그인 화면이 뜸
- 수집 광고 수가 갑자기 0으로 떨어짐

해결 순서:
1. **24시간 대기** — 메타 차단은 보통 시간이 지나면 자동 해제됩니다.
2. **본인 Chrome에서 광고 라이브러리 정상 접속** — 본인이 직접 광고 라이브러리에 들어가서 정상적으로 광고들이 보이는지 확인. 캡차 뜨면 풀고.
3. **임시 프로필 폴더 삭제** — `rm -rf /tmp/ad-scraper-profile` (다음 실행에서 본인 Chrome 프로필 새로 복사)
4. **테스트 한 번** — `python scraper/main.py --mode relevancy_monthly_grouped`를 경쟁사 1곳으로만 줄여서 (config.json 임시 수정) 시도.
5. **그래도 안 되면** — 일주일 쉬고 재시도. 또는 클로드한테 "차단당했어, 봇 시그널 더 줄일 방법 추가해줘"라고.

## 6. 자주 막히는 부분

### "Chrome 프로필을 못 찾았다"
- 본인 Chrome 프로필 경로가 다를 수 있습니다.
- `config.json`의 `chrome_profile_path`를 본인 환경에 맞춰 수정.
- Chrome이 깔려있지 않으면 폴백 모드로 일반 Chromium 사용 (단, 로그인 안 된 세션).

### "Playwright 브라우저가 안 열린다"
```bash
playwright install chromium --force
playwright install-deps  # Linux에서만 필요
```

### "수집은 되는데 캡션이 비어있다"
메타가 GraphQL 응답 구조 바꿨을 가능성. `scraper/extractor.py`의 `_node_to_ad` 함수가 시도하는 필드명을 늘려야 합니다. 클로드한테:

> `scraper/extractor.py`의 `_node_to_ad`가 캡션을 못 뽑아오는 광고가 많아. `data/ads.db`의 `raw_json` 컬럼 샘플 한 개 보고 새 필드명 후보 추가해줘.

### "영상이 다운로드 안 된다"
- 메타 영상 CDN URL은 토큰 만료가 있어서 응답 받자마자 다운로드해야 합니다.
- `scraper/media_download.py`는 즉시 다운로드하지만, 가끔 만료된 URL이 응답에 들어옵니다. 무시하고 다음 실행 때 다시 시도.

### "비용 걱정"
- 이 도구는 **외부 결제 0원**입니다 (Apify 같은 외부 서비스 안 씀).
- 본인 컴퓨터 전기료 + 저장 공간만 듭니다.
- 1,000개 광고 누적 시 ~5GB 정도 (영상 포함). 60일 지난 비활성 광고는 미디어만 삭제하는 정리 스크립트 추후 추가 예정.

이걸 어디에 끼울지:
