# 워크샵 가이드 — 클로드 코드랑 같이 만들어보기

> **이 가이드의 핵심: 아래 "그대로 복사할 프롬프트"를 클로드 코드 입력창에 붙여넣기만 하면 됩니다.**
> 직접 코드를 칠 필요도, 개발 용어를 외울 필요도 없어요. 우리는 "무엇을 원하는지"만 또박또박 말하면 됩니다.

---

## 0. 준비 (처음 한 번)

용어부터 가볍게:
- **터미널**: 컴퓨터에 글자로 명령하는 검은 창. (맥: `터미널` 앱)
- **클로드 코드**: 그 터미널 안에서 우리 대신 코드를 읽고 고쳐주는 AI 비서.
- **대시보드**: 모은 광고를 눈으로 보는 웹 화면.

### 0-1. 폴더 열기
다운로드한 폴더(`class101-ref-collector-workshop`)를 터미널에서 엽니다. 잘 모르겠으면, 터미널에 `cd ` (한 칸 띄우고) 입력 후 폴더를 창에 끌어다 놓고 엔터.

### 0-2. 한 번만 설치하기
아래를 터미널에 **한 줄씩** 붙여넣고 엔터:
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```
> `.venv`는 "이 프로젝트 전용 짐가방" 같은 거예요. 다른 프로그램과 안 섞이게 따로 담아두는 것.

### 0-3. 대시보드 켜보기 (연습용 광고 40개가 들어있어요)
```bash
streamlit run dashboard/app.py
```
브라우저가 열리면서 광고들이 보이면 성공! (안 열리면 http://localhost:8501)

> 💡 **더 쉬운 방법(더블클릭)**: 맥은 `START-Mac.command`, 윈도우는 `START-Windows.bat` 파일을
> 더블클릭하면 0-2·0-3(설치+대시보드 켜기)을 자동으로 해줍니다.
> (맥에서 경고가 뜨면 우클릭 → 열기. 윈도우 보호 화면은 "추가 정보 → 실행".)

## 토큰을 아끼는 5가지 습관 (중요!)

클로드 코드는 쓸수록 '토큰'(=사용량=비용)이 듭니다. 아래만 지켜도 절반 이하로 줄어요.

1. **한 번에 하나만 부탁하기.** "이것도 저것도" 말고 한 가지씩.
2. **어디를 고칠지 콕 집어주기.** 아래 프롬프트엔 파일·함수 이름이 이미 들어있어요. 그대로 쓰세요.
3. **"전부 다시 설명해줘" 금지.** 클로드가 길게 요약하려 하면 "설명 말고 코드만"이라고 하세요.
4. **새 채팅을 너무 자주 열지 않기.** 한 작업은 한 대화에서 끝내기 (맥락을 다시 안 읽어도 되게).
5. **`/clear` 활용.** 한 단계 끝나면 `/clear`로 대화를 비우고 다음 단계 시작 (이전 내용까지 매번 안 읽게).

> 이 폴더엔 `CLAUDE.md`가 있어서, 클로드 코드가 구조를 **다시 뒤지지 않아도** 됩니다. 그게 이미 토큰을 크게 아껴줘요.

## 1단계 [필수] — 사이드바 필터 살리기

**지금 상태**: 대시보드는 켜지지만, 왼쪽 사이드바의 필터(활성/영상/최소 게재일)를 움직여도 목록이 안 변해요.

**우리가 원하는 것**: 슬라이더·버튼을 움직이면 그 조건에 맞는 광고만 남기.

### 👉 클로드 코드에 그대로 붙여넣기
```
dashboard/app.py 파일의 apply_filters 함수를 완성해줘.
지금은 그냥 ads를 다 돌려주고 있어. 아래 조건대로 걸러서 돌려주게 만들어줘:
- active_filter 가 "활성만"이면 활성 광고만, "비활성만"이면 비활성만, "전체"면 통과
- media_filter 가 "이미지"면 이미지 광고만, "영상"이면 영상만, "전체"면 통과
- min_active_days 보다 적게 게재된 광고는 빼기
- start, end 가 있으면 그 게재시작일 범위 안의 광고만
함수 안만 고치고, 다른 파일이나 다른 함수는 건드리지 마. 설명은 짧게.
```

### 확인하는 법
1. 대시보드 화면으로 가서 (안 떠 있으면 다시 `streamlit run dashboard/app.py`)
2. 왼쪽 사이드바에서 **"최소 게재일"** 슬라이더를 오른쪽으로 옮겨보기 → 광고 개수가 줄어들면 ✅
3. **"영상"** 만 골라보기 → 영상 광고만 남으면 ✅

> 잘 안 되면 클로드 코드에: `필터가 동작 안 해. 슬라이더를 올려도 목록이 그대로야. 왜 그런지 보고 고쳐줘.`

## 2단계 [보너스/심화] — 광고 알맹이 뽑아내기 (자동화의 심장)

> 1단계만 해도 충분히 '맛보기' 완성이에요. 더 깊이 가보고 싶은 분만 도전하세요.

**배경**: 진짜로 경쟁사 광고를 긁어오면, 받아온 덩어리에서 캡션·이미지·게재일을 **뽑아내야** 합니다. 이 "뽑아내기"가 자동화에서 가장 자주 고장나는 부분이에요(메타가 구조를 바꾸니까). 그래서 이걸 클로드와 고치는 연습이 제일 실전적입니다.

**지금 상태**: 광고 번호만 뽑고 나머지는 비어 있음. 인터넷 없이 테스트할 수 있게 진짜 광고 응답 샘플 1건을 넣어뒀어요.

### 먼저 현재 상태 확인 (인터넷 불필요)
```bash
python scripts/test_extractor.py
```
캡션·소재·게재일이 ❌ 로 나오면 정상(아직 안 만든 상태).

### 👉 클로드 코드에 그대로 붙여넣기
```
scraper/extractor.py 의 _node_to_ad 함수를 완성해줘.
지금은 ad_id, page_id, page_name 만 뽑고 나머지는 비어 있어.
data/samples/sample_ad_response.json 이 진짜 광고 응답 1건이니, 그 구조를 보고
아래를 snapshot/node 에서 뽑아내게 채워줘:
- caption (본문 글), cta_text (버튼 문구), landing_url (클릭 시 가는 주소)
- media_urls : [{"type":"image" 또는 "video", "url": 주소}] 목록 (영상은 preview 주소도)
- start_date, active_days (며칠째 게재중인지), platforms, has_video
다 만들면 python scripts/test_extractor.py 를 돌려서 전부 ⭕ 가 되는지 확인해줘.
이 함수만 고치고 설명은 짧게.
```

### 확인하는 법
```bash
python scripts/test_extractor.py
```
캡션·소재·게재일이 ⭕ 로 바뀌고 맨 아래 `🎉 완성!` 이 뜨면 성공!
